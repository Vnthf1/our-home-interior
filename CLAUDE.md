# 우리집 인테리어 — 프로젝트 컨텍스트

두 사람이 함께 살 집의 리모델링 계획을 업체·지인과 공유하기 위한 **정적 웹사이트**.
공사 일정·공정별 작업 계획·컨셉·견적·연락처·도면을 한곳에 모았다.

## 기술 스택 / 구조

- **순수 정적 사이트** — 빌드 과정 없음. HTML + CSS + 바닐라 JS.
- 외부 의존성: Pretendard 폰트(CDN)뿐. 프레임워크·번들러 없음.
- 페이지마다 HTML은 **껍데기**이고, 실제 내용은 전부 JS가 그려 넣는다.

```
data.js     ← 모든 콘텐츠(데이터)가 여기 있음. 보통 이 파일만 고치면 됨.
app.js      ← 공통 렌더링 로직. data.js를 읽어 각 페이지 DOM을 생성.
styles.css  ← 전체 스타일. :root 에 색상 변수(따뜻한 베이지/브라운 톤).
images/     ← 도면·사진. data.js에서 파일명으로 참조.
quotes/     ← 견적서 파일(이미지/PDF).
*.html      ← 페이지 껍데기. <body data-page="..."> + 빈 컨테이너 div만 있음.
```

### 페이지 목록 (app.js의 NAV와 일치)
홈(index) · 컨셉(concept) · 공정표(schedule) · 작업계획서(plans) ·
작업안내(work) · 도면(floorplan) · 견적/후보(quotes) · 레퍼런스(references) · 연락처(contacts)

- **work.html(작업 안내)** = 작업자 공유용 진입점. 자체 헤더 + 탭바(공유 site nav 미사용). `PHASES`를 그대로 재사용하되 담당 업체명·견적·연락처는 숨김. 첫 탭은 개요(`OVERVIEW` + `BEFORE_AFTER`), 이후 공정별 탭. **탭마다 URL이 해시로 구분**(`work.html#electric`)되어 작업자에게 해당 공정만 공유 가능.
- plans.html(작업계획서)와 work.html은 `phaseCardHTML()`을 공유 → 작업계획서 포맷을 바꾸면 양쪽에 동시 반영. work.html은 `{hideTeam:true}`로 호출.

### 작업계획서 포맷 (PHASES) — 견적용 새 포맷
- `highlights`: 견적용 요약(맨 위 📌 박스). `[{ value:"규모/개수", label:"항목", note:"보충·확정필요" }]`.
- `groups[].items`: 문자열이거나 객체. 객체면 `{ text, undecided?:true, memo?:"직접구매 등", caution?:"주의점"|[...], ask?:"질문"|[...] }`.
  - `undecided` → **미정** 배지(점선 밑줄). `memo` → 옆에 **📝 메모** 태그(직접구매 등).
  - `caution` → **⚠️ 공정 시 주의**(주황 박스), `ask` → **💬 업자 확인**(파랑 박스). **둘은 별개 필드.**
  - ⚠️ caution·ask·memo는 **사용자가 직접 채운다. 임의로 넣지 말 것.**
- `summary`: 있으면 한 줄 표시, 비우면("") 아예 안 나옴(전기는 비워둠).
- `decisions`(⚠️ 결정 필요)·`checks`(✓ 확인)는 공정 단위 그대로 유지.
- **진행 현황**: 전기(electric)에 새 포맷 적용 완료(파일럿). 나머지 공정은 아직 문자열 items → 공정별로 하나씩 highlights/ask/undecided 채워가면 됨.

## 콘텐츠 수정 방법 (대부분 data.js만)

| 바꿀 것 | data.js의 위치 |
|---------|----------------|
| 프로젝트 정보·소개 | `PROJECT` |
| 컨셉/무드/마감재 | `CONCEPT` |
| 공정 일정(간트) | `SCHEDULE.tasks` (spans = [시작,종료] 날짜) |
| 공정별 작업계획 | `PHASES` (각 공정의 highlights/groups/decisions/checks/images) |
| 도면 마커 | `FLOORPLAN.items` |
| 레퍼런스 사진 | `REFERENCES` |
| 견적/후보 업체 | `QUOTES` (공정별 후보·견적) |
| 연락처 | `CONTACTS` |

이미지는 `images/`에 넣고 data.js에서 **파일명으로** 참조한다(예: `평면도.png`).

## 도면 페이지 (floorplan.html) — 인터랙티브 평면도

평면도(`images/평면도.png`) 위에 가구·조명·콘센트·스위치·메모 마커를 올린다.
- **보기**: 레이어 칩 토글, 마커 클릭 시 설명 팝업.
- **✏️ 편집 모드**: 클릭으로 추가(점/영역/글자), 드래그 이동, 영역 크기조절, 라벨/레이어 수정, 삭제.
- 편집 내용은 브라우저 localStorage에 임시저장 → **[📋 코드 복사]** 로 나온 `items:` 코드를
  `data.js`의 `FLOORPLAN.items`에 붙여넣어야 **모두에게 영구 반영**된다(백엔드 없음).
- 좌표는 이미지 대비 **%**(반응형). 마커 타입: `pin`(x,y) · `box`(x,y,w,h) · `text`(x,y).

## 견적/후보 페이지 (quotes.html) — 공정별 견적 정리

아직 확정 안 된 공정의 후보 업체와 받은 견적을 **공정별로** 모은다.
- `QUOTES` = `[{ phase, candidates: [...] }]`. `phase`는 `PHASES`의 id(아이콘·이름 자동), 렌더는 PHASES 순서.
- 후보 1건 형태: `{ name, company, phone, price, status, scope, summary, items:[{label,amount}], files, note }`.
  - `status`: `"decided"`(✅확정) · `"received"`(📄견적받음) · `"candidate"`(후보). 후보 0건이면 "🔍 후보 찾는 중".
  - `scope`: 📐 견적 **포함범위**(예: "시공비만" vs "자재·도기 포함"). 범위가 다른 견적을 헷갈리지 않게 하는 핵심 필드 — **금액 단순 비교/합산 금지**, scope로 구분.
  - `files`: 견적서 파일 경로 배열(`"quotes/xxx.pdf"` 또는 `{file,label}`). 이미지=썸네일, PDF=📄카드, 클릭 시 새 탭.
- **확정**: 전기(승구기사)·목공(이세환). 그 외는 견적/후보 단계.
- 견적서를 받으면 파일은 `quotes/`에 넣고, 내용을 이름·가격·세부내역(items)으로 요약해 정리한다.

## 작업 시 주의

- 빌드/테스트 도구 없음. JS 수정 후 `node --check app.js data.js` 로 문법만 확인하면 됨.
- **Playwright/브라우저 확인은 하지 말 것** — 이 단순 정적 사이트에선 문법 체크로 충분(사용자 선호).
- 텍스트는 한국어. CSS는 `:root` 색상 변수와 기존 클래스 네이밍을 따른다.
- 사용자가 data.js 등을 직접 편집하는 경우가 잦으니, 수정 전 항상 최신 내용을 다시 읽을 것.
